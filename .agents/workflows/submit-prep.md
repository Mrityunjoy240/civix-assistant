# Title: Submission Preparation
# Description: Prepares the final submission — polishes the app, generates blog content.

Step 1: Run a final QA check using the qa-check skill.

Step 2: Take final screenshots of all core screens. Save to production_artifacts/screenshots/.

Step 3: Generate a blog post draft in production_artifacts/Blog_Draft.md. 
The blog must include:
- The problem and why this interpretation was chosen over alternatives
- 3-5 key architectural decisions made and the trade-off for each
- One thing that went wrong and how it was fixed
- What the agent did autonomously vs what required human intervention
- Screenshots embedded inline
Target: 600-900 words. Tone: technical but readable for a developer audience.

Step 4: Generate a LinkedIn post summary (max 300 words) in production_artifacts/LinkedIn_Post.md.
Include: what was built, one surprising decision, a call to action, relevant hashtags: #BuildwithAI #GoogleAntigravity #PromptWars #VibeCoding
